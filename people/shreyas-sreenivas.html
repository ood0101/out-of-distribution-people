The Spike — With Evidence

Shreyas built Exa’s web-scale vector database, which handles billions of 4,096-dimensional vectors with sub-100ms query latency and >500 queries/second throughput — using less memory than a gaming PC. The system is written in Rust and uses five key optimizations he and the team developed:

  • Matryoshka embedding training: Reduced 4,096-dim vectors to 256-dim prefixes — 20x memory reduction
  • Binary quantization: 16-bit floats compressed to 1-bit values — another 16x memory reduction
  • Lookup table optimization: Precomputed dot products for length-4 subvectors loaded into CPU registers — 4x compute reduction
  • K-means clustering: 100,000 clusters for ~1,000x throughput improvement vs. brute force
  • SIMD assembly optimizations: Hand-tuned low-level operations for maximum hardware utilization

Critically, Exa chose clustering over HNSW (the standard graph-based approach used by Pinecone, Weaviate, etc.), enabling easier data sharding, more efficient metadata filtering (date, domain, keyword), and a 10x cost reduction versus cloud vector database services. This is an architectural bet that requires deep systems knowledge to execute — not a wrapper around an existing library.

Additionally, Exa migrated their highlights/embedding server from Python to Rust (Feb 2024) to serve real-time embeddings at scale, and built custom BM25 serving with 50% memory reduction using a novel encoding scheme (May 2025). Shreyas’s skill set — Rust, CUDA, Go, distributed systems — maps directly onto these infrastructure challenges.
