What Judgment Labs Is

Tagline: “The continuous-improvement stack for agents.” Self-described as “an applied-research lab solving last-mile agent reliability in production.” The four product surfaces:

  • Agent Search — behavioral-level query across agent trajectories (not just trace search)
  • Agent Judge — create trajectory-level evaluators using harnesses (the design primitive)
  • Behavior Discovery — identify failure modes from unlabeled production data
  • AutoRubrics — auto-construct evaluation rubrics from verifiable signals

Workflow: Slack alert when agent misbehaves → agent-swarm triage finds similar failures and narrows root cause → testing framework validates fix against production cases → deploy. The differentiator vs LangSmith / Braintrust / Arize is that the product is framed around shipping fixes, not building dashboards. The judgeval README explicitly positions itself as “the open source post-building layer for agents — our environment data and evals power agent post-training (RL, SFT) and monitoring.” That is the differentiated thesis: production data as fuel for RL/SFT improvement, not just observability.
